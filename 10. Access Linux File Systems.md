# Access Linux File Systems

This chapter covers methods for:  
  - Identifying file systems and devices
  - Mounting and unmounting file systems and devices
  - Locating files on the system

**Methods of Finding/Searching Files**
  There are two primary commands for finding files in Linux:  
    1. `locate` - Fast database search
    2. `find` - Real-time search with extensive criteria

---

**Searching Files Using `locate` Command**

  Search files that match the specified pattern in their name using a pre built database

- Characteristics:
  - Fast (uses database)
  - Searches entire system
  - Database must be updated regularly with `updatedb` (usually runs via cron)
  - Simple pattern matching
  - Case insensitive by default

- Syntax:
```bash
locate <pattern>
```

- Examples:
```bash
locate test
locate passwd
locate "*.conf"
```

- Update database manually:
```bash
sudo updatedb
```

---

**Searching Files Using `find` Command**

- Search files based on different criteria (name, ownership, permission, size, type, etc.) in real time
- Use the `find` command with the root user for comprehensive system wide searches
```bash
su - root
# OR
sudo find ... 
```
  
- Syntax:
```bash
find <path> [expression]
```

- Expression format:
```bash
find <path> [tests] [actions]
```

- Search Criteria Options

| Option | Description |
|--------|-------------|
| `-name <pattern>` | Name based search (case sensitive) |
| `-iname <pattern>` | Name based search (case insensitive) |
| `-atime <+N/-N/N>` | Access-time based search (days) |
| `-mtime <+N/-N/N>` | Modification time based search (days) |
| `-ctime <+N/-N/N>` | Change time based search (days) |
| `-size <+N/-N/N>` | Size based search |
| `-perm <mode>` | Permission based search |
| `-user <owner>` | User ownership based search |
| `-group <group>` | Group ownership based search |
| `-type <type>` | File type based search |
| `-empty` | Empty files or directories |
| `-newer <file>` | Files newer than specified file |



- Find Command Examples:

- Case 1: Time Based Searches

- Time Notation

| Notation | Meaning |
|----------|---------|
| `-3` | Within last 3 days (less than 3 days old) |
| `+3` | Before 3 days (more than 3 days old) |
| `3` | Exactly 3 days ago (between 72-96 hours ago) |


```bash
# Find files modified in the last 3 days
find /etc -mtime -3

# Find files modified exactly 3 days ago
find /etc -mtime 3

# Find files modified more than 3 days ago
find /etc -mtime +3

# Find files accessed in last 3 days
find /etc -atime -3

# Find files changed (metadata) in last 7 days
find /etc -ctime -7

# Find files modified in last 24 hours
find /var/log -mtime 0

# Find files accessed in last 60 minutes
find /tmp -amin -60

# Find files modified in last 2 hours
find /var -mmin -120
```

Case 2: Name Based Finding

> IMPORTANT: Quote Wildcards!

```bash
# CORRECT - Quotes prevent shell expansion
find /etc -name "*.conf"
find /etc -name '*.conf'

# WRONG - Shell expands * before find runs
find /etc -name *.conf
```

- Examples

```bash
# Search files with .conf extension in the entire system
find / -name "*. conf"

# Search .conf files only in /etc directory
find /etc -name "*.conf"

# Case insensitive search
find /etc -iname "*. CONF"
find /etc -iname "*.conf"

# Count .conf files (case sensitive)
find /etc -name "*.conf" | wc -l

# Count .conf files (case insensitive)
find /etc -iname "*.conf" | wc -l

# Find files starting with "host"
find /etc -name "host*"

# Find files containing "network" in name
find /etc -name "*network*"

# Find exact filename
find /etc -name "passwd"

# Find multiple patterns using -o (OR)
find /etc \( -name "*.conf" -o -name "*.cfg" \)
```

**Searching Files Based on Multiple Criteria**

- Logical Operators

| Operator | Alternative | Function | Description |
|----------|-------------|----------|-------------|
| `-and` | (implicit) | Logical AND | Both conditions must be true |
| `-or` | `-o` | Logical OR | At least one condition must be true |
| `-not` | `!` | Logical NOT | Negates the condition |
| `\( \)` | | Grouping | Groups conditions (must be escaped) |

> Note:`-and` is the default operator and is usually implicit between tests. 

- Examples

```bash
# Find .conf files with permission 770 (implicit -and)
find /etc -name "*.conf" -perm 770

# Explicit -and (same as above)
find /etc -name "*.conf" -and -perm 770

# Find .conf files with permission 600
find /etc -name "*.conf" -perm 600

# Find .conf files OR files with permission 600
find /etc -name "*.conf" -o -perm 600
find /etc -name "*.conf" -or -perm 600  # Same

# Find .conf files but NOT with permission 600
find /etc -name "*.conf" !  -perm 600
find /etc -name "*.conf" -not -perm 600  # Same

# Multiple criteria:  .conf files with permission 600 and size > 1KB
find /etc -name "*.conf" -perm 600 -size +1k

# Complex grouping:  Find .jpg OR .png files larger than 1MB
find ~/Pictures \( -name "*.jpg" -o -name "*.png" \) -size +1M

# Find files modified today but NOT owned by root
find /var/log -mtime 0 !  -user root

# Find . log OR .txt files modified in the last 7 days
find /var \( -name "*.log" -o -name "*.txt" \) -mtime -7
```

- Verify Results

```bash
# Check permissions
ls -ld <path_of_one_file>

# Check size in human readable format
ls -lh <path_of_one_file>

# Check all attributes
stat <path_of_one_file>
```

**Finding and Processing Files**

  - Perform actions on found files using `-exec`, `-ok`, or `-delete`

- Syntax:
```bash
find <path> <criteria> -exec <command> {} \;
find <path> <criteria> -exec <command> {} +
find <path> <criteria> -ok <command> {} \;
```

- Command Components

| Component | Description |
|-----------|-------------|
| `-exec command {} \;` | Execute command once per file (slower) |
| `-exec command {} +` | Execute command with multiple files (faster) |
| `-ok command {} \;` | Execute command with confirmation per file |
| `{}` | Placeholder for each found file |
| `\;` | Marks end of -exec command (must be escaped) |
| `+` | Passes multiple files to command at once |


- Performance Comparison
```
-exec rm {} \;     Runs:   rm file1; rm file2; rm file3
-exec rm {} +      Runs:   rm file1 file2 file3
```

> Use `+` when possible for better performance!


- Finding and Processing Examples

- Example 1: Remove Empty Files

```bash
# Find and remove empty files in the current directory (with confirmation)
find . -size 0c -ok rm {} \;

# Find and remove empty files (without confirmation)
find . -size 0c -exec rm {} \;

# Better:  Use -delete (more efficient)
find . -size 0c -delete

# Find and remove empty files in /root
find /root -size 0c -delete

# Remove empty directories
find /tmp -type d -empty -delete
```

- Example 2: List Large Files

```bash
# Find files bigger than 10KB and list with sizes
find /var/log -size +10k -exec ls -lh {} \;

# More efficient (passes multiple files at once)
find /var/log -size +10k -exec ls -lh {} +

# Find and sort by size
find /var/log -size +10k -exec ls -lh {} + | sort -k5 -h
```

- Example 3: Compress Large Files

```bash
# Find files bigger than 10KB and compress them
find /var/log -size +10k -name "*.log" -exec gzip {} \;

# Find and compress with confirmation
find /var/log -size +10k -name "*.log" -ok gzip {} \;

# Find uncompressed log files older than 7 days and compress
find /var/log -name "*.log" -mtime +7 -exec gzip {} \;

# Compress with progress (one by one)
find /var/log -size +1M -exec sh -c 'echo "Compressing:  $1"; gzip "$1"' _ {} \;
```

- Example 4: Backup Configuration Files

```bash
# Create backup directory
mkdir -p /root/confbackup

# Find .conf files and copy (files only)
find /etc -name "*.conf" -type f -exec cp {} /root/confbackup \;

# Better: Use + for efficiency
find /etc -name "*. conf" -type f -exec cp {} /root/confbackup \; 

# Find .conf files and copy with directory structure
find /etc -name "*. conf" -exec cp --parents {} /root/confbackup \;

# Create dated backup
mkdir -p /backup/$(date +%Y%m%d)
find /etc -name "*.conf" -type f -exec cp {} /backup/$(date +%Y%m%d)/ \;
```

> Note: The `-r` flag is not needed with `find` because it only processes files it finds.

- Example 5: Find Files in Size Range

```bash
# Find files between 10KB and 15KB and list them
find /etc -size +10k -size -15k -exec ls -lh {} +

# Find files between 1MB and 10MB
find /var -size +1M -size -10M -type f -exec ls -lh {} +

# Find and count files in size range
find /home -size +100M -size -500M | wc -l
```

- Example 6: Change Permissions

```bash
# Find and change permissions for directories
find /var/www/html -type d -exec chmod 755 {} +

# Find and change permissions for files
find /var/www/html -type f -exec chmod 644 {} +

# Find world-writable files and fix them
find /home -perm -002 -exec chmod o-w {} +

# Find and change ownership
find /var/www -type f -exec chown www-data: www-data {} +
```

- Example 7: Copy with Rename

```bash
# Find and copy with .bak extension
find .  -name "*.conf" -exec cp {} {}.bak \;

# Find and move files
find /tmp -name "*.tmp" -mtime +7 -exec mv {} /tmp/archive/ \;
```

- Example 8: Execute Complex Commands

```bash
# Find and execute multiple commands
find /var/log -name "*.log" -size +100M -exec sh -c 'echo "Processing:  $1"; gzip "$1"; mv "$1.gz" /archive/' _ {} \;

# Find and create checksums
find /etc -name "*.conf" -exec md5sum {} + > /root/checksums.txt

# Find and display with custom format
find /home -type f -exec stat --format='%n %s bytes' {} \;
```

- Example 9: Delete Old Files

```bash
# Delete files older than 30 days
find /tmp -type f -mtime +30 -delete

# Delete log files older than 90 days (with confirmation)
find /var/log -name "*.log" -mtime +90 -ok rm {} \;

# Delete files older than 1 year and log the action
find /archive -type f -mtime +365 -exec sh -c 'echo "Deleting: $1" >> /var/log/cleanup.log; rm "$1"' _ {} \;
```

- Example 10: Find and Archive

```bash
# Find and create tarball
find /etc -name "*.conf" -type f | tar -czf /backup/configs.tar.gz -T -

# Find modified today and archive
find /var/log -mtime 0 -type f -exec tar -rvf /backup/today.tar {} +
```

**Advanced Examples and Use Cases**

- Use Case 1: Security Auditing

```bash
# Find SUID/SGID binaries (potential security risks)
find / -type f \( -perm -4000 -o -perm -2000 \) -exec ls -lh {} + 2>/dev/null

# Find world-writable files and directories
find / \( -type f -o -type d \) -perm -002 !  -type l -ls 2>/dev/null

# Find files with no owner or group
find / \( -nouser -o -nogroup \) -exec ls -l {} + 2>/dev/null

# Find files modified in last 24 hours in system directories
find /etc /bin /sbin /usr/bin /usr/sbin -mtime 0 -ls 2>/dev/null

# Find files accessible by others in home directories
find /home -type f -perm -004 -exec ls -l {} + 2>/dev/null
```

- Use Case 2: System Cleanup

```bash
# Clean old temporary files
find /tmp -type f -atime +7 -delete
find /var/tmp -type f -mtime +30 -delete

# Remove old log files
find /var/log -name "*.log.*" -mtime +90 -delete

# Clean package manager cache
find /var/cache/apt/archives -name "*.deb" -mtime +30 -delete

# Remove core dump files
find / -name "core" -type f -exec rm {} + 2>/dev/null

# Clean thumbnail cache
find ~/. cache/thumbnails -type f -atime +30 -delete
```

- Use Case 3: Disk Space Analysis

```bash
# Find the top 20 largest files
find / -type f -exec du -h {} + 2>/dev/null | sort -rh | head -20

# Find the largest files in /var
find /var -type f -size +100M -exec ls -lh {} + 2>/dev/null | sort -k5 -rh

# Find directories consuming the most space
find /home -type d -exec du -sh {} + 2>/dev/null | sort -rh | head -10

# Find duplicate files by size
find /home -type f -exec du -b {} + | sort -n | uniq -d -w 10

# Find files larger than available space
df -h / | awk 'NR==2 {print $4}' # Check available space
find / -type f -size +10G -exec ls -lh {} + 2>/dev/null
```

- Use Case 4: Development and Build

```bash
# Find and remove compiled files
find .  \( -name "*.o" -o -name "*.pyc" -o -name "*. class" \) -delete

# Find source files modified today
find .  \( -name "*.c" -o -name "*.h" \) -mtime 0 -exec ls -l {} +

# Find large binary files in repository
find .  -type f -size +1M !  -path "*/.git/*" -exec ls -lh {} +

# Find files without proper extension
find /var/www -type f !  -name "*.*" -exec file {} + | grep text

# Count lines of code
find .  -name "*.py" -type f -exec wc -l {} + | awk '{sum+=$1} END {print sum}'
```

- Use Case 5: Backup and Archival

```bash
# Incremental backup (files modified in last 24 hours)
find /home -type f -mtime 0 -exec cp --parents {} /backup/daily/ \;

# Find and backup with compression
find /etc -name "*.conf" -type f | tar -czf /backup/configs-$(date +%Y%m%d).tar.gz -T -

# Find and sync to remote server
find /data -type f -mtime -1 -exec rsync -avz {} user@server:/backup/ \;

# Create backup excluding certain files
find /home/user -type f !  -name "*.mp4" ! -name "*.iso" | tar -czf /backup/user. tar.gz -T -
```

- Use Case 6: Log File Management

```bash
# Rotate large log files
find /var/log -name "*.log" -size +100M -exec sh -c 'mv "$1" "$1.old"; > "$1"' _ {} \;

# Find and compress old logs
find /var/log -name "*.log" -mtime +7 !  -name "*.gz" -exec gzip {} \;

# Extract errors from all log files
find /var/log -name "*.log" -exec grep -H "ERROR" {} + > /tmp/all_errors.txt

# Count entries in all log files
find /var/log -name "*. log" -exec sh -c 'echo "$1: $(wc -l < "$1")"' _ {} \;
```

- Use Case 7: File System Maintenance

```bash
# Find and fix permissions for web directories
find /var/www -type d -exec chmod 755 {} +
find /var/www -type f -exec chmod 644 {} +

# Find and remove broken symbolic links
find /usr/local -type l !  -exec test -e {} \; -delete

# Find files with spaces in names and rename
find .  -name "* *" -type f -exec bash -c 'mv "$1" "${1// /_}"' _ {} \;

# Find duplicate filenames (case-insensitive)
find /home -type f -printf "%f\n" | tr '[: upper:]' '[:lower:]' | sort | uniq -d

# Verify file integrity
find /etc -name "*.conf" -type f -exec md5sum {} + > /tmp/integrity.md5
```

- Use Case 8: Media File Management

```bash
# Find and organize photos by date
find ~/Pictures -name "*.jpg" -exec sh -c 'DATE=$(date -r "$1" +%Y-%m); mkdir -p ~/Pictures/organized/$DATE; mv "$1" ~/Pictures/organized/$DATE/' _ {} \;

# Find large video files
find /home -type f \( -name "*.mp4" -o -name "*.avi" -o -name "*.mkv" \) -size +500M -exec ls -lh {} +

# Find duplicate images by size
find ~/Pictures -type f -name "*. jpg" -exec du -b {} + | sort -n | uniq -d -w 10

# Convert image formats
find .  -name "*.png" -exec sh -c 'convert "$1" "${1%.png}.jpg"' _ {} \;

# Find corrupted media files
find /media -type f \( -name "*.mp3" -o -name "*.mp4" \) -exec ffmpeg -v error -i {} -f null - \; 2>&1 | grep -v "Output file"
```

- Use Case 9: Database and Configuration

```bash
# Backup all configuration files modified today
find /etc -name "*. conf" -o -name "*.cfg" -mtime 0 -exec cp --parents {} /backup/configs/ \;

# Find configuration files with passwords (security audit)
find /etc -type f -exec grep -l "password" {} + 2>/dev/null

# Compare configurations across systems
find /etc -name "*.conf" -exec md5sum {} + | sort > /tmp/config_hashes.txt

# Find and validate XML/JSON configurations
find /etc -name "*.xml" -exec xmllint --noout {} \; 2>&1 | grep -v "validates"
find /etc -name "*.json" -exec python3 -m json. tool {} \; >/dev/null 2>&1
```

- Use Case 10: Network and Services

```bash
# Find socket files
find /var/run -type s -ls

# Find files accessed by specific service
find /var -user www-data -atime -1 -exec ls -lu {} +

# Find configuration files for running services
systemctl list-units --type=service --state=running --no-pager | awk '{print $1}' | while read svc; do find /etc -name "$svc*"; done

# Monitor file changes in real-time (requires inotify-tools)
find /var/www -type f | while read file; do inotifywait -m "$file"; done
```

## Practical Workflows

**Workflow 1: System Cleanup**

```bash
# Find and remove empty files
find /tmp -type f -empty -delete

# Find and remove old log files (older than 30 days)
find /var/log -type f -name "*.log" -mtime +30 -delete

# Find and compress large log files
find /var/log -type f -size +100M -name "*.log" !  -name "*.gz" -exec gzip {} \;

# Clean package cache
find /var/cache -type f -mtime +30 -delete 2>/dev/null
```

**Workflow 2: Configuration Backup**

```bash
# Create timestamped backup directory
BACKUP_DIR="/backup/configs/$(date +%Y%m%d_%H%M%S)"
mkdir -p "$BACKUP_DIR"

# Find and backup all . conf files with directory structure
find /etc -name "*.conf" -type f -exec cp --parents {} "$BACKUP_DIR" \;

# Create tarball
tar -czf "$BACKUP_DIR.tar.gz" -C /backup/configs "$(date +%Y%m%d_%H%M%S)"

# Verify backup
echo "Backup contains $(find "$BACKUP_DIR" -type f | wc -l) files"
```

**Workflow 3: Security Audit**

```bash
# Find world-writable files
echo "=== World-Writable Files ===" 
find / -perm -002 -type f !  -type l -ls 2>/dev/null > /tmp/security_audit.txt

# Find SUID/SGID files
echo "=== SUID/SGID Files ===" >> /tmp/security_audit.txt
find / \( -perm -4000 -o -perm -2000 \) -type f -ls 2>/dev/null >> /tmp/security_audit.txt

# Find files with no owner
echo "=== Files with No Owner ===" >> /tmp/security_audit.txt
find / \( -nouser -o -nogroup \) -ls 2>/dev/null >> /tmp/security_audit.txt

# Find recently modified system files
echo "=== Recently Modified System Files ===" >> /tmp/security_audit.txt
find /etc /bin /sbin -mtime -1 -ls 2>/dev/null >> /tmp/security_audit.txt

echo "Security audit complete. Results in /tmp/security_audit.txt"
```

**Workflow 4: Disk Space Analysis**

```bash
# Find largest files
echo "=== Top 20 Largest Files ==="
find / -type f -exec du -h {} + 2>/dev/null | sort -rh | head -20

# Find large files in /var
echo "=== Large Files in /var (>100MB) ==="
find /var -type f -size +100M -exec ls -lh {} + 2>/dev/null | sort -k5 -rh

# Find directories consuming most space
echo "=== Top 10 Largest Directories ==="
find /home -type d -exec du -sh {} + 2>/dev/null | sort -rh | head -10

# Summary
echo "=== Disk Usage Summary ==="
df -h
```

**Workflow 5: Development Cleanup**

```bash
# Remove compiled files and caches
find . -type f \( -name "*.pyc" -o -name "*. pyo" -o -name "__pycache__" \) -delete
find . -type f \( -name "*.o" -o -name "*.a" -o -name "*.so" \) -delete
find . -name "node_modules" -type d -exec rm -rf {} + 2>/dev/null
find . -name ".DS_Store" -delete

# Clean build artifacts
find . -name "build" -type d -exec rm -rf {} + 2>/dev/null
find . -name "dist" -type d -exec rm -rf {} + 2>/dev/null
find . -name "*. egg-info" -type d -exec rm -rf {} + 2>/dev/null

echo "Development cleanup complete"
```

**Find Command Decision Tree**

```
┌─────────────────────────────────────────────────────┐
│  What are you searching for?                        │
└─────────────────┬───────────────────────────────────┘
                  │
    ┌─────────────┼─────────────┬────────────┐
    │             │             │            │
    ▼             ▼             ▼            ▼
By Name      By Time      By Properties  By Content
    │             │             │            │
    │             │             ├─ Size     Use grep with
    │             │             ├─ Permission  find -exec
    │             │             ├─ Owner
    │             │             ├─ Group
    │             │             └─ Type
    │             │
    │             ├─ Modified (-mtime)
    │             ├─ Accessed (-atime)
    │             └─ Changed (-ctime)
    │
    ├─ Case sensitive (-name)
    └─ Case insensitive (-iname)

         ┌─────────────────────┐
         │ Need to process?      │
         └────────┬────────────┘
                  │
         ┌────────┴────────┬────────────┐
         │                 │            │
         ▼                 ▼            ▼
    With confirm      Without       Delete only
      (-ok)           (-exec)       (-delete)
```

**Common Problems and Solutions**

| Problem | Cause | Solution |
|---------|-------|----------|
| Permission denied errors | Insufficient privileges | Use `sudo` or `su -` and redirect errors:  `2>/dev/null` |
| Too many results | Search path too broad | Narrow search path or add more criteria |
| Slow searches | Searching entire filesystem | Limit search path; use `locate` for name searches |
| Accidental deletions | Using `-exec` carelessly | Always use `-ok` for destructive operations |
| Wildcards not working | Shell expansion | Quote wildcards:  `"*. conf"` not `*.conf` |
| Parentheses error | Shell interpretation | Escape parentheses: `\(` and `\)` |
| `-exec` not working | Missing `\;` or `+` | Always end with `\;` or `+` |
| Finding broken symlinks | Wrong syntax | Use:  `find / -type l !  -exec test -e {} \; -print` |
| Case sensitivity issues | Using `-name` instead of `-iname` | Use `-iname` for case-insensitive |
| Size notation confusion | Wrong units | Remember: `c`=bytes, `k`=KB, `M`=MB, `G`=GB |

**Performance Comparison**

```
┌──────────────────────────────────────────────────┐
│  locate vs find                                  │
├──────────────────────────────────────────────────┤
│                                                  │
│  locate:                                         │
│  ✓ Very fast (uses database)                     │
│  ✓ Simple syntax                                 │
│  ✗ Limited search criteria (name only)           │
│  ✗ Requires updated database (updatedb)          │
│  ✗ May show deleted files                        │
│  ✗ No real time results                          │
│                                                  │
│  find:                                           │
│  ✓ Real time results                             │
│  ✓ Extensive search criteria                     │
│  ✓ Can perform actions on results                │
│  ✓ Accurate (reflects current state)             │
│  ✗ Slower (searches filesystem)                  │
│  ✗ More complex syntax                           │
│                                                  │
└──────────────────────────────────────────────────┘
```
