## Creating a Virtual Machine on UTM (macOS), VirtualBox (Windows/Unix/Linux, and KVM/QEMU (Ubuntu)

Creating a new virtual machine (VM) on three common virtualisation platforms: 
  - UTM for macOS
  - Oracle VirtualBox for Windows and Unix/Linux
  - KVM/QEMU (via virt-manager) for Ubuntu

---

## Creating a VM using UTM on macOS

  - **Platform:** macOS (Apple Silicon or Intel)
  - **Prerequisites:** UTM application installed. Guest OS installation media (e.g., ISO file).

**Steps:**

1.  **Launch UTM:** Open the UTM application from your `Applications` folder or via Spotlight search.

2.  **Initialise VM Creation:**
    *   Click the "**Create a New Virtual Machine**" button.
      
<img width="547" height="600" alt="Screenshot 2025-11-28 at 9 31 45 AM" src="https://github.com/user-attachments/assets/9e2cdcc4-ebd7-4bb6-856f-2942ac83ddd8" />

3.  **Select Virtualisation Method:**
      * **Virtualize:** Use when guest and host architectures match for best performance.(e.g., ARM Linux/macOS on Apple Silicon)
      * **Emulate:** Use when architectures differ, with slower performance.(e.g., x86_64 Windows/Linux on Apple Silicon)

4.  **Allocate Hardware Resources:**
    *   **Memory:** 2048 MB RAM.
    *   **CPU Cores:** 2 CPU cores.
      
5.   **Configure the Operating System:**
    * Select the guest operating system type (e.g., Linux, Windows, macOS).
      * Linux
        * Boot Image Type: Boot from ISO image
        * Boot ISO Image
          * Click "**Browse...**" to select the guest OS installation ISO image.

6.  **Configure Storage:**
    * A minimum of **15-20 GB** is recommended.
      
7. Click "**Save**" to create the VM.
   
9. Select the created VM, Right click the VM, and then **Edit**.

    *  Network
      * Network Mode: Bridged Networking
      * MAC Address: Random (Click once in Random)

10.  **Finalise and Boot:**
    *   Click "**Save**"
    *   Click "**Play**" to power on the VM and begin the OS installation.

## Creating a VM using Oracle VirtualBox on PC

  - **Platform:** Windows, Linux, or macOS (x86)
  - **Prerequisites:** Oracle VM VirtualBox installed. Guest OS installation media.

**Steps:**

1.  Open the Oracle VM VirtualBox application.

2.  **Create a New VM:**
    *   Click the "**New**" button.
    *   VM Name: RedHat
    *   ISO Image: Click the dropdown menu and select 'Other', and select the downloaded ISO image file.
    *   OS: Linux
    *   OS Distribution: Red Hat
    *   OS Version: Red Hat 10.x

3.  **Allocate Hardware Resources:**
    *   Base Memory: 2048 MB
    *   CPUs: 2
    *   Disk Size: 15-20 GB
    *   Seeting
      * Network: Attached to `Bridged Adapter`  

4.   Click "**Finish**"
       - Click "**Start**" to power on the VM and begin the OS installation.

## Creating a VM using KVM/QEMU on Ubuntu

**Platform:** Ubuntu Linux (or similar distributions)
**Prerequisites:** `virt-manager` installed. User added to the `libvirt` group. Virtualisation is enabled in BIOS/UEFI.

**Steps:**

1.  **Install and Verify Environment:**
    *   Install the management GUI:
        ```bash
        sudo apt update && sudo apt install virt-manager
        ```
    *   Verify KVM support is enabled:
         > `sudo apt install cpu-checker`
        ```bash
        sudo kvm-ok
        ```
    *   Launch the application:
        ```bash
        virt-manager
        ```

2.  **Initialise VM Creation:**
    *   In `virt-manager`, click "**File > New Virtual Machine**" or the "**Create a new virtual machine**" button.

3.  **Select Installation Method:**
    *   Choose "**Local install media (ISO image or CDROM)**" 
    *   Click "**Browse...**" to locate and select your guest OS ISO file.
    *   The OS type and version should be auto detected. If not, select them manually.

4.  **Allocate Memory and CPU:**
    *   **Memory:** Allocate at least **2048 MB**.
    *   **CPUs:** Assign a minimum of **2** virtual CPUs.

5.  **Configure Storage:**
    *   Enable "**Enable storage for this virtual machine**."
    *   Create a new disk image with a minimum recommended size of **15-20 GB**.

6.  **Customise Configuration (Recommended):**
    *   Before starting the installation, check the box: "**Customise configuration before install**."
    *   Click "**Finish**" and review the following key settings in the new window:
        *   **Overview > Firmware:** Change to "**UEFI**" if required by the guest OS (e.g., Windows 11).
        *   **CPUs > Configuration:** Enable "**Copy host CPU configuration**" for optimal performance and compatibility.
        *   **NIC > Device Model:** For Linux guests, change to "**virtio**" for superior network performance.

7.  **Begin Installation:**
    *   Click "**Begin Installation**" to start the VM and proceed with the guest operating system installation.

**Things to consider:**

*   **Resource Allocation:** Always balance VM resource allocation with the host system's available resources to avoid host system degradation.
*   **VirtIO Drivers:** For KVM and UTM, using the VirtIO interface for storage and network provides significantly better I/O performance. Appropriate drivers may be required within the guest OS.
  
*   **Installation Media:** Ensure the integrity of the OS ISO file before beginning the process to avoid installation failures.
  
*   **Guest Additions/Tools:** After the guest OS is installed, always install the respective hypervisor's guest additions (VirtualBox) or QEMU guest agent (KVM/UTM) to enable features like shared folders, better video support, and seamless mouse integration.
