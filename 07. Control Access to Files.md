# Control Access to Files

This chapter covers: 
- Interpreting Linux file system permissions
- Managing file system permissions from the command line
- Managing default permissions and file access

---

**Types of Users (Ownership Classes)**

Linux recognizes three classes of users for file permissions:

| Class | Symbol | Description |
|-------|--------|-------------|
| **User (Owner)** | `u` | The user who owns the file |
| **Group** | `g` | Members of the file's group |
| **Others** | `o` | All other users on the system |
| **All** | `a` | All three categories (u+g+o) |

> Note: If you don't specify a class, `chmod` uses **all** (`a`) as the default.

---

**Types of Permissions**

- Permission Types

| Permission | Symbol | Numeric Value | Effect on File | Effect on Directory |
|------------|--------|---------------|----------------|---------------------|
| **Read** | `r` | 4 | View file contents | List directory contents |
| **Write** | `w` | 2 | Modify file contents | Create/delete files in directory |
| **Execute** | `x` | 1 | Run file as program | Enter directory (cd) |
| **No permission** | `-` | 0 | No access | No access |


**Permission Representation Methods**

I. Symbolic Permission

Uses letters:  `r` (read), `w` (write), `x` (execute), `-` (no permission)

- Format: `rwxrwxrwx`
  - First triplet: Owner permissions
  - Second triplet: Group permissions
  - Third triplet: Others permissions

- Example: `rwxrw-r--`
  - Owner: `rwx` (read, write, execute)
  - Group: `rw-` (read, write)
  - Others: `r--` (read only)

II. Numeric (Octal) Permission

Uses numbers 0-7, calculated by adding permission values: 

| Numeric | Binary | Symbolic | Calculation | Permissions |
|---------|--------|----------|-------------|-------------|
| 0 | 000 | `---` | 0+0+0 | No permissions |
| 1 | 001 | `--x` | 0+0+1 | Execute only |
| 2 | 010 | `-w-` | 0+2+0 | Write only |
| 3 | 011 | `-wx` | 0+2+1 | Write and execute |
| 4 | 100 | `r--` | 4+0+0 | Read only |
| 5 | 101 | `r-x` | 4+0+1 | Read and execute |
| 6 | 110 | `rw-` | 4+2+0 | Read and write |
| 7 | 111 | `rwx` | 4+2+1 | Read, write, and execute |

- Example: `755`
  - Owner: `7` = `rwx` (4+2+1)
  - Group: `5` = `r-x` (4+0+1)
  - Others: `5` = `r-x` (4+0+1)

---

**Types of Operators for Changing Permissions**

| Operator | Function | Description |
|----------|----------|-------------|
| `=` | Assign | Set exact permissions (overwrites existing) |
| `+` | Add | Add permissions to existing ones |
| `-` | Remove | Remove permissions from existing ones |

### Operator Examples

**Case I: Existing permission is `rw-` (6)**

```bash
# Adding execute permission
chmod +x file     # Result: rwx (7)

# Setting only execute permission (overwrites)
chmod =x file     # Result: --x (1)
```

## 5. Effects of Permissions

### 5.1 Effects on Files

| Permission | Effect |
|------------|--------|
| `r` (Read) | View file contents with `cat`, `less`, `more`, etc. |
| `w` (Write) | Modify or delete file contents |
| `x` (Execute) | Run file as a program or script |

**Important Notes:**
- **Write permission alone doesn't allow editing** if read permission is missing (you can't see what you're editing)
- **Execute permission is only meaningful for scripts and programs**

### 5.2 Effects on Directories

| Permission | Effect |
|------------|--------|
| `r` (Read) | List directory contents with `ls` |
| `w` (Write) | Create, delete, or rename files in directory |
| `x` (Execute) | Enter directory with `cd` and access files within |

**Important Notes:**
- **Execute permission is required to `cd` into a directory**
- **Write permission allows modifying directory contents (creating/deleting files)**
- **Read permission alone doesn't allow `cd` without execute**

## 6. Viewing Permissions

### 6.1 View File/Directory Permissions

```bash
# View file permissions
ls -l filename

# View directory permissions
ls -ld dirname

# View multiple files
ls -l file1 file2
```

### 6.2 Output Format

```
<type><permissions> <links> <owner> <group> <size> <date/time> <name>
```

**Example:**
```
-rwxrw-r-- 1 david marketing 1024 Jan 15 10:30 file1
```

### 6.3 Permission Field Format (10 characters)

```
- rwx rw- r--
│ │   │   │
│ │   │   └── Others permissions (3 chars)
│ │   └────── Group permissions (3 chars)
│ └────────── Owner permissions (3 chars)
└──────────── File type (1 char)
```

### 6.4 File Type Characters

| Character | File Type | Description |
|-----------|-----------|-------------|
| `-` | Regular file | Normal file (text, binary, etc.) |
| `d` | Directory | Directory/folder |
| `l` | Symbolic link | Shortcut to another file |
| `c` | Character device | Character device file (e.g., `/dev/tty`) |
| `b` | Block device | Block device file (e.g., `/dev/sda`) |
| `p` | Named pipe | FIFO (First In, First Out) pipe |
| `s` | Socket | Local socket file |

**View device files:**
```bash
cd /dev
ls -l
```

## 7. Permission Examples Explained

### Example 1: Regular File
```
-rwxrw-r-- 1 david marketing 1024 Jan 15 10:30 file1
```

**Breakdown:**
- **File type:** `-` (regular file)
- **Owner:** `david` with `rwx` (7) - read, write, execute
- **Group:** `marketing` with `rw-` (6) - read, write
- **Others:** `r--` (4) - read only
- **Numeric equivalent:** `764`

### Example 2: Directory
```
drwxr-xr-x 2 root root 4096 Jan 15 10:30 /etc/config
```

**Breakdown:**
- **File type:** `d` (directory)
- **Owner:** `root` with `rwx` (7) - full access
- **Group:** `root` with `r-x` (5) - read and enter
- **Others:** `r-x` (5) - read and enter
- **Numeric equivalent:** `755`

## 9. Permission Change Scenarios

### Scenario 1: Full Control for Owner, Read for Group, None for Others

**Requirement:**
- Owner: `rwx` (7)
- Group: `r--` (4)
- Others: `---` (0)

#### Method I: Symbolic Method
```bash
chmod u=rwx,g=r,o= file1
ls -l file1
```

#### Method II: Numeric Method
```bash
chmod 740 file2
ls -l file2
```

---

### Scenario 2: Read/Write for Owner and Group, Read for Others

**Requirement:**
- Owner: `rw-` (6)
- Group: `rw-` (6)
- Others: `r--` (4)

#### Symbolic Method (Option 1)
```bash
chmod u=rw,g=rw,o=r file1
```

#### Symbolic Method (Option 2 - Combined)
```bash
chmod ug=rw,o=r file1
```

#### Numeric Method
```bash
chmod 664 file2
```

#### Apply to Multiple Files
```bash
# Symbolic
chmod ug=rw,o=r file1 file2

# Numeric
chmod 664 file1 file2
```

---

### Scenario 3: Read and Execute for All Users

**Requirement:**
- Owner: `r-x` (5)
- Group: `r-x` (5)
- Others: `r-x` (5)

#### Symbolic Method (Option 1)
```bash
chmod u=rx,g=rx,o=rx file1
```

#### Symbolic Method (Option 2 - All specified)
```bash
chmod ugo=rx file1
```

#### Symbolic Method (Option 3 - All shorthand)
```bash
chmod a=rx file1
```

#### Numeric Method
```bash
chmod 555 file2
```

## 10. Adding/Removing Permissions (Incremental Changes)

**Purpose:** Modify existing permissions without overwriting them completely. 

### 10.1 Examples

```bash
# Create test file
vim file3
ls -l file3

# Add execute permission for owner
chmod u+x file3

# Add execute for group, read/write/execute for others
chmod g+x,o+rwx file3

# Remove read/write from group, read/write from others
chmod g-rw,o-rw file3

# Remove execute from all users
chmod a-x file3

# Add execute to all users
chmod a+x file3
```

### 10.2 Summary of Operators

| Operator | Effect | Example | Result |
|----------|--------|---------|--------|
| `+` | Add permission | `chmod u+x file` | Adds execute for owner |
| `-` | Remove permission | `chmod g-w file` | Removes write from group |
| `=` | Set exact permission | `chmod o=r file` | Sets others to read only |

**Important Note:** Numeric method **always overwrites** - it doesn't support `+` or `-` operators. Numeric is like an assignment operator.

## 11. Directory Permissions

### 11.1 Setting Directory Permissions

```bash
# Create directory with file
mkdir newdir
cd newdir
vim f1
cd ..

# View directory permissions
ls -ld newdir

# Change directory permissions
chmod 700 newdir
ls -ld newdir
```

**Important:** Permission changes on a directory **only affect the directory itself**, not its contents.

### 11.2 Recursive Permission Change

**Use `-R` flag to apply permissions to directory and all its contents:**

```bash
chmod -R 700 newdir
ls -ld newdir
cd newdir
ls -l
```

**Both `newdir` and `f1` will have the same permissions.**

## 12. Verifying Effects of Permissions

### 12.1 Effects on Files

#### Test 1: No Permissions (000)
```bash
# Create file with content
vim f1
# Write some content

# Remove all permissions
chmod 000 f1
ls -l f1

# Try to read - FAILS
cat f1    # Permission denied
```

#### Test 2: Read Permission Only
```bash
# Add read permission for owner
chmod u+r f1

# Read succeeds
cat f1

# Try to write - FAILS
vim f1
# Write content
: wq       # Readonly option is set
: q! 
```

#### Test 3: Write Permission Without Read
```bash
# Add write permission only
chmod u+w f1
chmod u-r f1

# Try to edit
vim f1
# Content won't be displayed for editing
```

**Important:** Even with write permission, you need read permission to view and edit file contents.

#### Test 4: Write Permission Only (200)
```bash
vim f2
# Write content

chmod 200 f2

# Try to edit
vim f2
# Contents will not be displayed
```

**Note:** Execute permission on regular files only matters for executable files (programs/scripts).

### 12.2 Effects on Directories

#### Test 1: No Permissions (000)
```bash
mkdir dir
chmod 000 dir
ls -ld dir

# Try to enter - FAILS
cd dir    # Permission denied
```

#### Test 2: Execute Permission Only (100)
```bash
# Add execute permission
chmod u+x dir

# Enter succeeds
cd dir

# Try to list - FAILS
ls        # Cannot open directory:  Permission denied
```

#### Test 3: Execute and Read (500)
```bash
cd .. 
chmod u+r dir

cd dir
ls        # Now works

# Try to create file - FAILS
touch f1      # Permission denied
mkdir dir1    # Permission denied
```

**Note:** Without write permission, cannot create, delete, or rename files. 

#### Test 4: Full Permissions (700)
```bash
cd ..
chmod u+w dir

cd dir
ls

# All operations now work
touch f1
mv f1 file
mkdir dir2
rm file
```

## 13. umask - Default Permissions

### 13.1 What is umask?

**umask** determines the default permissions assigned to files and directories when they are created.

### 13.2 View Current umask

```bash
umask
# Output: 0002
```

**Format:** `0002`
- First digit (`0`): Special permissions (SUID, SGID, Sticky bit)
- Last three digits (`002`): Permission mask for User, Group, Others

**Note:** Leading zero can be ignored for general permissions.

### 13.3 Default Permissions with umask

```bash
# Login as normal user
vim file1
mkdir dir1

# Check default permissions
ls -l file1    # -rw-rw-r-- (664)
ls -ld dir1    # drwxrwxr-x (775)
```

**Note:** Root user and normal users typically have different umask values, resulting in different default permissions.

## 14. Calculating Default Permissions

### 14.1 Default File Permissions

**Formula:**
```
Maximum allowed permission for files (666)
- umask
──────────────────────────────────────────
Default permission on file
```

#### Example 1: Normal User (umask = 002)
```
  6 6 6    (maximum for files)
- 0 0 2    (umask)
─────────
  6 6 4    = rw-rw-r--
```

#### Example 2: Root User (umask = 022)
```
  6 6 6    (maximum for files)
- 0 2 2    (umask)
─────────
  6 4 4    = rw-r--r--
```

### 14.2 Default Directory Permissions

**Formula:**
```
Maximum allowed permission for directories (777)
- umask
──────────────────────────────────────────
Default permission on directory
```

#### Example 1: Normal User (umask = 002)
```
  7 7 7    (maximum for directories)
- 0 0 2    (umask)
─────────
  7 7 5    = rwxrwxr-x
```

#### Example 2: Root User (umask = 022)
```
  7 7 7    (maximum for directories)
- 0 2 2    (umask)
─────────
  7 5 5    = rwxr-xr-x
```

### 14.3 Common umask Values

| umask | File Default | Directory Default | Use Case |
|-------|--------------|-------------------|----------|
| 0000 | 666 (rw-rw-rw-) | 777 (rwxrwxrwx) | Very permissive (insecure) |
| 0002 | 664 (rw-rw-r--) | 775 (rwxrwxr-x) | Normal user default |
| 0022 | 644 (rw-r--r--) | 755 (rwxr-xr-x) | Root user default |
| 0027 | 640 (rw-r-----) | 750 (rwxr-x---) | Group-shared |
| 0077 | 600 (rw-------) | 700 (rwx------) | Private/secure |

## 15. Changing umask Value

### 15.1 Syntax

```bash
umask <new_value>
```

### 15.2 Scenario Example

**Requirement:** Owner gets read and write permission, but group and others get no permissions on files at creation.

**Desired result:** `-rw-------` (600)

**Calculate umask:**
```
  6 6 6    (maximum for files)
- 6 0 0    (desired result)
─────────
  0 6 6    (required umask)
```

### 15.3 Method I:  Temporary Change

```bash
# Open terminal
umask 066

# Verify
umask
# Output: 0066

# Create test file
vim testfile
# Write content

# Check permissions
ls -l testfile
# Output: -rw------- (600)

# Close and reopen terminal
umask
# Output: 0002 (default restored)
```

**Note:** Temporary change lasts only for the current shell session.

### 15.4 Method II: Permanent Change

#### Case I: User-Specific (Current User Only)

```bash
# Edit user's . bashrc
vim ~/.bashrc

# Add line
umask 066

# Save and exit
:wq

# Apply changes
source ~/.bashrc

# Verify
umask
# Output:  0066
```

#### Case II: System-Wide (All Users)

```bash
# Switch to root
su - root

# Edit system-wide bashrc
vim /etc/bashrc

# Add line
umask 066

# Save and exit
: wq

# Switch to another user to test
su - david

# Open terminal
umask
# Output: 0066
```

## 16. Special Cases with Wildcards

### 16.1 Execute Permission on All Files

```bash
# Create multiple files and directories
mkdir b1 b2 b3 b4
touch f1 f2 f3

# Set initial permissions
chmod 700 b1 b2 b3 b4 f1 f2 f3

# Add execute for others on everything
chmod o+x *

# Check permissions
ls -l
# Both files AND directories have execute permission
```

**Note:** `*` matches all contents in current directory.

### 16.2 Selective Execute Permission (Directories Only)

```bash
# Remove execute from others
chmod o-x *

# Add execute ONLY to directories (capital X)
chmod o+X *

# Check permissions
ls -l
# Only directories have execute permission
```

**Important:** 
- Lowercase `x`: Adds execute to ALL files/directories
- Uppercase `X`: Adds execute ONLY to directories (or files that already have execute for some user)

### 16.3 Difference Between x and X

| Option | Effect |
|--------|--------|
| `chmod +x` | Adds execute to all files and directories |
| `chmod +X` | Adds execute only to directories and executables |

**Example:**
```bash
# Create test environment
mkdir testdir
touch regularfile
touch script.sh
chmod u+x script.sh

# Apply uppercase X
chmod a+X *

ls -l
# testdir:  execute added
# regularfile: NO execute added
# script.sh: execute added (already had it for user)
```

## 17. Comprehensive Examples

### Example 1: Secure Personal Directory

```bash
# Create personal directory
mkdir ~/private

# Set secure permissions (owner only)
chmod 700 ~/private

# Create files inside
cd ~/private
touch document.txt
ls -l document.txt
# Default: -rw-rw-r-- (depends on umask)

# Set secure umask
umask 077

# Create new file
touch secure.txt
ls -l secure.txt
# Output: -rw------- (600)
```

### Example 2: Shared Project Directory

```bash
# Create project directory
mkdir /projects/team_project

# Set group collaboration permissions
chmod 770 /projects/team_project

# Set appropriate umask for team members
umask 007

# Create files in project
cd /projects/team_project
touch roadmap.txt
mkdir docs

ls -l
# roadmap.txt: -rw-rw---- (660)
# docs: drwxrwx--- (770)
```

### Example 3: Public Read-Only Directory

```bash
# Create public directory
mkdir /public/docs

# Set read-only for all
chmod 755 /public/docs

# Create files inside
cd /public/docs
touch readme.txt

# Set read-only permissions
chmod 644 readme.txt

ls -l
# readme.txt: -rw-r--r-- (644)
# docs: drwxr-xr-x (755)
```

### Example 4: Web Server Directory

```bash
# Create web directory
mkdir -p /var/www/html/website

# Set directory permissions
chmod 755 /var/www/html/website

# Set file permissions recursively
find /var/www/html/website -type f -exec chmod 644 {} +
find /var/www/html/website -type d -exec chmod 755 {} +

# Verify
ls -lR /var/www/html/website
# Directories: drwxr-xr-x (755)
# Files: -rw-r--r-- (644)
```

### Example 5: Executable Script

```bash
# Create script
vim backup.sh
#!/bin/bash
echo "Backup script"

# Make executable for owner only
chmod 700 backup.sh

# Or make executable for all users
chmod 755 backup.sh

# Verify
ls -l backup.sh
./backup.sh    # Execute script
```

## 20. Troubleshooting Common Issues

| Problem | Cause | Solution |
|---------|-------|----------|
| Can't read file | No read permission | `chmod u+r file` |
| Can't edit file | No write permission | `chmod u+w file` |
| Can't execute script | No execute permission | `chmod u+x script` |
| Can't cd into directory | No execute permission | `chmod u+x directory` |
| Can't list directory | No read permission | `chmod u+r directory` |
| Can't create file in directory | No write permission on directory | `chmod u+w directory` |
| Vim shows "readonly" | No write permission | `chmod u+w file` or use `:w!` in vim (if you own the file) |
| Script runs but can't see output | Script lacks read on output file | Check script and output file permissions |

## 21. Key Takeaways

1. **Three permission types:** Read (r/4), Write (w/2), Execute (x/1)
2. **Three user classes:** Owner (u), Group (g), Others (o), All (a)
3. **Two representation methods:** Symbolic (rwx) and Numeric (0-7)
4. **Three operators:** = (set), + (add), - (remove)
5. **Files vs Directories:** Permissions have different meanings
6. **Execute on directories:** Required to access directory contents
7. **Write needs read:** To edit files effectively, you need both permissions
8. **umask:** Determines default permissions at file/directory creation
9. **Calculate defaults:** Subtract umask from maximum (666 for files, 777 for dirs)
10. **Uppercase X:** Adds execute only to directories (not regular files)
11. **Recursive changes:** Use `-R` flag carefully
12. **Security principle:** Grant minimum necessary permissions
13. **Common patterns:** 644 (files), 755 (dirs/scripts), 600 (private), 700 (private dirs)
14. **Temporary umask:** Lasts current session only
15. **Permanent umask:** Set in `~/.bashrc` (user) or `/etc/bashrc` (system-wide)

## 22. Command Summary

```bash
# View permissions
ls -l file                    # View file permissions
ls -ld directory              # View directory permissions

# Change permissions (symbolic)
chmod u=rwx,g=rx,o=r file    # Set exact permissions
chmod u+x file                # Add execute for owner
chmod go-w file               # Remove write from group and others
chmod a=r file                # Set read-only for all

# Change permissions (numeric)
chmod 755 file                # rwxr-xr-x
chmod 644 file                # rw-r--r--
chmod 600 file                # rw-------

# Recursive changes
chmod -R 755 directory        # Apply to directory and contents

# View umask
umask                         # Show current umask

# Change umask
umask 022                     # Temporary
echo "umask 022" >> ~/.bashrc # Permanent (user)

# Special cases
chmod o+X *                   # Execute on directories only
```
